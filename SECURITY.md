# Security Policy

## Supported versions

These packages are pre-1.0 (`0.x`). Security fixes land on the latest published version
of each `@trembus/*` package; please upgrade to the newest release before reporting.

## Reporting a vulnerability

Please **do not** open a public issue for security vulnerabilities.

Instead, report privately via one of:

- [GitHub private vulnerability reporting](https://github.com/nicholasosto/Trembus-Component-Library/security/advisories/new)
  (Security → Report a vulnerability), or
- email **nicholas.osto@gmail.com** with a description and reproduction steps.

You'll get an acknowledgement as soon as possible. Once a fix is available, a patched
release is published and the advisory disclosed. Thanks for helping keep the library and
its users safe.
